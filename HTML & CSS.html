HTML and CSS

HTML and CSS are coding languages used to render a page and develop a website to be added to the Internet. This website, coincidently using HTML and CSS, will explain the basics of what these terms mean.

What is HTML and why is it important?

  • What does it stand for?
    HTML stands for HyperText Markup Language, and it is the language behind creating a webpage. It can be seen as the language that the browser can only understand.

  • How is it used?
    HTML is how you can "talk" to the browser so your webpage will turn out how you wish. The language consists of tags in between left and right angle brackets with shortened, limited letters commands to specify how the page should be displayed.

  • Where did it come from?
    Tim Berners-Lee invented HTML, or more generally the Web, with a means to pooling together documents and information from other scientists around the world using hypertext to connect those documents and create a "web" of information in computers across the globe.

  • How is it used?
    HTML is used by adding whatever content you wish to input in between angle bracketed tags dictating how the content should be displayed or rendered by the page. For example, to write a paragraph with HTML, use the tags "p" and "/p" within left and right angle brackets and write whatever you want in between the two tags. Some tags are used to tell the page how the information should look on the page, like its file name or tab title, or decide how the page should be described/labeled as.

What is CSS and why is it important?

  • What does it stand for?
    CSS stands for Cascading style sheets, and it is the coding language used to style and design your page to add that uniqueness to it, as well as manage the layout of the page.

  • How is it used?
    CSS also uses tags, either within HTML tags or under a tag called "style" and "/style". It requires more spaced out options like dictating a certain section of the page that requires styling, adding a sguiggly bracket, hitting "Enter", typing in the styling option, then typing in how the style should be, and ending with another sguiggly bracket. For every styling option, from background images to font size, there has to be a series of descriptions with ranges of numbers and details.

  • Where did it come from
    CSS was first proposed by Hakon Wium Lie, co-worker of Tim-Berners Lee, in October 10, 1994. It was built upon over the months following with a series of proposals for different style sheets, collaborations, presentations, and more work, but eventually CSS became official at the beginning of 1997.

  • How is it written?
    As mentioned before, CSS uses tags, but in different places than HTML tags. CSS tags can be implemented with the beginning tag of an HTML tag couple, as well as have its own "style" and "/style" tag within the "head" and "/head" tags.

How do tags work?

  • What is a tag?
    • Tags are like the hidden keywords that help the browser decide how to display content.

  • How do write one? All tags require a left angle bracket, whatever keyword you wish, and then a right andle bracket. After writing down your content, if not done already, you need another tag with the same brackets and keywords, except for a backlash inputed before the keyword.

What are atrributes and how do they work?

  • How do you change attributes?
    Attributes are modifying keywords added to the start tags that can further specify how content should be displayed.

  • What are some examples?
    Some examples of attributes are the "src" attribute, which specifies a url for an image; the "style" attribute, which allows CSS actions within HTML; and the "href" attribute, which adds detail for the link "a" tag about which url should the "a" tag lead to.

Descriptions of at least 4 tags.

  • "head"
    The "head" tag basically defines information about a page, like its background title, its internal CSS, or how the page allows a separate tab all about CSS to connect to it.

  • "style"
    The "style" tag allows for CSS styling options within the HTML tags without the need to edit CSS on a separate sheet.

  • "ul"
    The "unordered list" tag starts a bullet point list, followed by "list" ("li") tags and the content in between.

  • "h1 - h6"
    The tags "h1" through "h6" are heading tags based on importance, where "h1" is most important and "h6" is of lowest importance.
